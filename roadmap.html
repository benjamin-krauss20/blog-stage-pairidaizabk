Fonctionnalités manquantes
Brancher le formulaire de contact Facile

Le formulaire n'envoie rien actuellement. Formspree règle ça sans aucun backend, en 5 minutes.

Voir comment faire

1. Créer un compte gratuit sur formspree.io.

2. Créer un nouveau form → tu récupères un endpoint : https://formspree.io/f/xabcdefg.

3. Dans index.html, modifier la balise form : <form action="https://formspree.io/f/TONID" method="POST">.

4. Supprimer le novalidate et le handler JS de submit — Formspree gère la redirection et l'email automatiquement.

Ajouter une favicon Facile

L'onglet du navigateur est vide. Le logo Pairi Daiza en favicon ferait très propre.

Voir comment faire

Dans le <head> de index.html, ajouter :

<link rel="icon" type="image/png" href="photo/logo.png">

Si le logo est sur fond blanc, générer une vraie .ico avec favicon.io pour un meilleur rendu sur tous les navigateurs.

Rédiger les 4 articles du journal Moyen

Les liens "Lire l'article" pointent tous vers #. C'est le vrai contenu du blog — la priorité absolue.

Voir comment faire

Option simple : créer une page HTML par article (article-glpi.html, article-zabbix.html…) avec le même style CSS que le blog.

Option avancée : utiliser des modales — au clic, une overlay s'ouvre avec le contenu sans quitter la page.

Structure suggérée pour chaque article : contexte du projet → problème rencontré → solution mise en place → ce que j'ai appris.

Astuce : si tu bloques sur la rédaction, demande à Morgane de t'aider à structurer le contenu à partir de tes notes de stage.

Mettre une vraie photo de toi dans "About" Moyen

La section About utilise une photo du zoo. Une vraie photo de toi en situation renforce l'aspect personnel.

Voir comment faire

Remplacer photo/entrezoom.png dans la section .about-img-wrap par ta photo.

Format recommandé : orienté portrait, environ 400×500px, JPEG compressé à 80%.

Une photo en situation (au bureau ou dans le parc) est plus percutante qu'un fond neutre. Évite les selfies.

Qualité & performances
Optimiser les images en WebP Facile

Les JPEG du carousel sont probablement lourds. Les convertir en WebP réduit le poids de 30 à 50%.

Voir comment faire

Outil gratuit recommandé : squoosh.app — drag & drop, exporte en WebP à 80% de qualité.

Ajouter aussi loading="lazy" sur toutes les balises <img> sauf le hero (qui doit se charger immédiatement).

Exemple : <img src="photo/ours.webp" loading="lazy" alt="Ours">

Ajouter les balises Open Graph Moyen

Sans ça, quand tu partages le lien sur LinkedIn l'aperçu sera vide. À faire avant toute publication.

Voir comment faire

Ajouter ces 4 lignes dans le <head> de index.html :

<meta property="og:title" content="Benjamin Krauss | Stage Pairi Daiza">

<meta property="og:description" content="Blog de stage — Administrateur Systèmes & Réseaux">

<meta property="og:image" content="https://blog-stage-pairidaizabk.vercel.app/photo/entre.png">

<meta property="og:url" content="https://blog-stage-pairidaizabk.vercel.app">

Tester le résultat sur opengraph.xyz après le déploiement.

Séparer CSS et JS en fichiers externes Avancé

Tout est dans index.html (1261 lignes). Séparer en style.css et main.js c'est une bonne pratique pro.

Voir comment faire

1. Couper tout le contenu de <style>...</style> → le coller dans style.css.

2. Dans le <head>, remplacer par : <link rel="stylesheet" href="style.css">.

3. Couper le JS interne → le coller dans main.js, puis ajouter <script src="main.js"></script> juste avant </body>.

⚠️ Attention : les fichiers style.css et main.js existent déjà dans le repo — vérifier leur contenu avant d'écraser.

Ajouter Plausible Analytics Avancé

Savoir combien de recruteurs ou de profs visitent le blog — c'est motivant et ça s'apprend vite.

Voir comment faire

Utiliser Plausible — respectueux de la vie privée, pas de RGPD à gérer, gratuit 30 jours.

Une seule ligne à ajouter dans le <head> :

<script defer data-domain="blog-stage-pairidaizabk.vercel.app" src="https://plausible.io/js/script.js"></script>

Bugs mobile — à corriger
Hamburger mal positionné sur mobile Moyen

Le bouton hamburger s'affiche en dessous du sélecteur de langue au lieu d'être à côté. Revoir l'alignement du header-right sur mobile.

Voir le contexte

Le .header-right contient le lang switcher et le hamburger. Sur mobile, ils s'empilent verticalement au lieu d'être côte à côte.

Fix : vérifier que .header-right a bien display: flex; align-items: center; gap: 12px; et que rien ne force un flex-direction: column sur mobile.

Le drawer mobile ne s'ouvre pas Moyen

Le clic sur le hamburger transforme l'icône en croix mais le panel de navigation ne slide pas. Le drawer reste invisible.

Voir le contexte

Le #mobileDrawer reçoit bien la classe is-open (la croix apparaît), mais le panel ne s'affiche pas.

Pistes : vérifier que .mobile-drawer.is-open { display: block; } est bien appliqué, que le z-index du panel est suffisant, et que l'animation drawerSlideDown ne cache pas le panel pendant son exécution.

Alternative fiable : remplacer display: none / block par visibility: hidden / visible + pointer-events: none / auto pour éviter les conflits avec les animations CSS.
